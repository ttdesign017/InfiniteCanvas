Windows 优先、本地优先、可嵌套、擅长混合媒体与精细摆放的创意灵感白板。

IC2 baseline

从代码确认的产品能力

以下不是路线图描述，而是当前仓库中已经存在的能力。它们构成本次竞品对比的基线。

CANVAS CORE

无限画布与高精度空间操作

平移、缩放、框选、多选、旋转感知命中、组合缩放、边缘吸附、对齐、压紧布局，以及 G / R / S 模态变换。整体明显强于常规便签白板。

MEDIA

混合媒体

图片、GIF、视频、自由文字、卡片笔记、链接预览、涂鸦与网页嵌入。

NESTING

嵌套画布

Stack 不是视觉分组，而是可进入的子画布，支持多层路径、折叠预览与跨层复制。

REFERENCE FLOW

参考图工作流

拖放与剪贴板导入、媒体裁剪和还原、自然尺寸恢复、置顶层级与批量布局。

LOCAL FIRST

便携项目文件

.icanvas 将素材内嵌为 base64，可离线打开，并兼容旧版 JSON。

DESKTOP

Windows 原生桌面体验

Tauri 2 + WebView2，支持文件关联、系统对话框、无边框窗口、关闭保存保护与本地文件读写。当前没有账号、云同步与多人协作依赖。

CURRENT GAPS

尚未覆盖

全局搜索、标签、浏览器采集器、PDF 与音频原生阅读、模板、评论、实时协作、跨设备同步、移动端，以及面向素材的 AI 检索。

Code review

先提升作品可信度，再扩展功能面。

生产构建已通过。以下按用户数据风险、故障影响与后续开发成本排序，只列值得行动的结论。

P0 · 数据

保存过程不是原子写入

当前直接覆盖目标文件，随后再回读校验。校验可以发现截断，却无法在进程崩溃、磁盘写满或写入中断时恢复原文件。

建议：写入同目录临时文件，校验成功后执行原子替换，并保留一个滚动备份。核心位置为 boardIO.ts:57。

P1 · 稳定

大项目文件会产生极高内存峰值

文件先整体读成字符串，再做 JSON 解析，然后把内嵌 base64 展开为 data URL。512 MiB 的上限并不等于安全，实际峰值可能是文件大小的数倍。

建议：读取前检查文件元数据并先降到保守上限；中期将项目格式改为 zip 容器或二进制资产包，素材按需解码。位置为 boardIO.ts:115 与 boardFile.ts:329。

P1 · 回归

复杂几何与嵌套状态缺少前端自动化测试

useCanvasStore.ts 超过 4,100 行，InfiniteCanvas.tsx 接近 2,900 行，但 package 中没有前端测试脚本。旋转、裁剪、历史记录、嵌套迁移和保存格式都容易出现组合回归。

建议：先为纯函数补 Vitest，再覆盖 5 条端到端主链路：导入、裁剪、嵌套、撤销、保存重开。Rust 当前仅有 2 项链接安全测试。

P1 · 安全

任意网页嵌入仍拥有较宽权限

iframe 同时允许脚本、同源、表单与弹窗，CSP 又允许任意 http(s) frame 和连接。对个人本地工具尚可接受，但分发给更广泛用户前需要更明确的信任模型。

建议：默认采用域名白名单和低权限 sandbox，特殊播放器按 provider 提权；对未知来源显示一次性风险确认。位置为 embedIframeCache.ts:13 与 tauri.conf.json:31。

P2 · 无障碍

图标按钮主要依赖 title，状态语义不足

左侧工具和多数右侧操作没有显式 aria-label，活动工具也未提供 aria-pressed。画布虽可聚焦，但缺少角色和可读名称；关闭对话框未见焦点圈定。

建议：补充可访问名称与状态，进入对话框时聚焦主按钮，Tab 限制在对话框内，Escape 取消。位置为 Toolbar.tsx:149 与 CloseSaveDialog.tsx:22。

P2 · 架构

核心状态与交互组件已到拆分临界点

构建后的主 JS 为 414.85 kB，gzip 128.32 kB。构建还提示两个动态导入因已有静态导入而无法形成分包。更重要的是，巨型 store 与巨型 canvas 组件降低修改可验证性。

建议：按 document、selection、viewport、stack、history 切分 store actions；把手势状态机移到独立模块。移除无效动态导入，并同步修正 README 中已过时的 “CSP 为 null” 描述。

Market landscape

竞品不是同一条赛道，而是四种用户任务。

IC2 最合理的位置在“私人深度创作”与“视觉参考工作台”的交界处。团队协作白板应视为替代品，而非功能清单模板。

Allume

原名 Muse · 最接近

allume.com

核心能力

嵌套画板、文字、手写、卡片、图片、视频、PDF、网页链接、文件、Inbox、连接线、搜索跳转、PDF 标注、PDF / PNG 导出、本地优先同步。Pro 提供共享工作区、实时光标、评论与模板，新版加入 MCP / AI 支持。

与 IC2 对比

Allume 强：Apple Pencil、PDF、搜索、Inbox、跨设备同步、协作、评论。
IC2 强：Windows、自由缩放与旋转、媒体裁剪、批量对齐压紧、PureRef 式精细操作、便携单文件。

FREE：2 个工作区，每区 100 卡片，1 位协作者 · UNLIMITED：$9.99/月 或 $99.99/年

PureRef

参考图板 · 次接近

pureref.com

Strategic direction

建议路线：可信、采集、检索。

这三个阶段会让 IC2 更像一个完整产品，同时继续放大已有差异，而不是进入团队白板的功能军备竞赛。

先守住作品

把“本地优先”从技术形态变成可感知的信任承诺。

  • 原子保存与自动备份
  • 大文件预检和恢复模式
  • 核心手势、嵌套与文件格式测试
  • 嵌入来源安全分级

再缩短采集路径

Allume 的 Inbox 与 Milanote、Eagle 的 Web Clipper 都在解决同一个摩擦。

  • 全局收件箱和快速投递
  • 浏览器扩展或系统分享入口
  • PDF 与音频原生卡片
  • 保留原始 URL、作者与采集时间

最后让素材可复用

当画布增多后，检索价值会超过新增一种卡片类型。

  • 全局搜索、标签与筛选
  • 颜色、文件类型和来源检索
  • 重复素材与相似图发现
  • 可选的本地 OCR / 语义检索
