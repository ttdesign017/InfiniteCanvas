# 发版前手测清单（白话版）

自动检查（`npm run check`）盖不住「手感、窗口、真实文件」时，用本清单。  
**发版或大改交互后**：先本地 `npm run check` 全绿，再勾 P0。

> 用打包版或 `npm run tauri dev` 测完整能力（浏览器 `npm run dev` 没有原生保存/部分桌面能力）。

---

## 每次发版必过（P0）

勾选前在干净状态开始（可新建板或已知测试板）。

| # | 做什么 | 期望结果 | 通过 |
|---|--------|----------|------|
| 1 | 新建板 → 拖入/打开 **图、视频、音频、文本、链接** → `Ctrl+S` 保存 → **完全退出再打开**该文件 | 内容都在，位置大致一致，无丢媒体 | ☐ |
| 2 | 多选 free 项 → `Ctrl+G` 建 stack → **双击进入** → 里面挪几下 → `Esc` 退出 | 能进能出；退出后 fan/文件夹看起来正常，可再进入 | ☐ |
| 3 | 做出 **A⊃B⊃C**（或三层嵌套）→ 进入最内层 → 点 breadcrumb **跳到上层/Home** | 各层 fan 不错位；还能再点进去 | ☐ |
| 4 | 选中图/文本 → `G` 移动 / `R` 旋转 / `S` 缩放 → **LMB 确认** 与 **Esc/RMB 取消** 各试一次 → `Ctrl+Z` | 确认后保留；取消回到变换前；撤销正常 | ☐ |
| 5 | 未旋转图按住 `C` 框选裁剪 → `Alt+C` 还原；再 **先旋转** 再试 `C` | 能裁能还原；旋转后裁被拒绝且有提示，图不被弄坏 | ☐ |
| 6 | 改一点内容（变脏）→ 关窗口或 `Ctrl+Q` | 出现 Save / Discard / Cancel；三种结果符合预期 | ☐ |
| 7 | 试打开：**坏 JSON**、**空文件**、或明显不是工程的文件（可选：超大文件） | 有错误提示；**当前已打开的板不被换成半残数据** | ☐ |
| 8 | 若已安装并关联 `.icanvas`：资源管理器 **双击** 工程文件 | 应用启动并打开该板 | ☐ |

P0 全勾才建议对外发安装包 / portable。
