## 0. 产品意图（先校准再比竞品）

### 0.1 你在做的不是「艺术参考板」 alone

结合当前代码能力与产品方向，更准确的意图是：

| 维度 | 方向 |
|------|------|
| **灵感搜集** | 图、视频、链接、摘录、网页嵌入，随手丢上画布 |
| **写作 / 混排表达** | 粘贴文字与图片、笔记卡、自由文本，图文同屏 |
| **Research / 研判** | 选品、趋势、对照、聚类；空间排布代替文件夹树 |
| **组织哲学** | **自由二维空间**（非瀑布流、非时间线、非强制网格）→ 邻近、叠放、缩放、嵌套催生新知 |

一句话：

> **本地优先的空间化灵感与研究桌面**  
> （Spatial canvas for inspiration, writing fragments, and research — free layout, nested structure, mixed media.）

绘画参考（PureRef 主场景）只是相邻能力，**不是唯一北极星**。  
Muse 一类「thinking canvas」与 Freeform 一类「系统白板」的心智，与本产品**同样相关甚至更相关**。

### 0.2 明确不做什么（边界）

| 不做 / 弱做 | 原因 |
|-------------|------|
| 社交剪藏流（Pinterest / Cosmos 类） | 瀑布流与内容形态单一，**无法自由空间排布**；与产品哲学冲突 |
| 企业实时协作白板（Miro 主场） | 另一类产品与组织形态 |
| 完整 DAM 素材库（Eagle 级） | 检索与库规模工程巨大；可互补而非取代 |
| 宿主内绘画取色工具链 | 可后置，非核心 |

### 0.3 核心用户叙事（跨行业）

不只艺术：

- **产品 / 选品 / 电商**：竞品截图、详情页、价格带、笔记并置  
- **研究 / 咨询 / 趋势**：文献摘录、图表、视频、链接预览同一画布  
- **写作 / 内容**：素材碎片 + 段落草稿空间化大纲  
- **设计 / 品牌**：情绪板，但带文字与链接深度  
- **个人知识**：比笔记软件更「看见全局」，比白板更「媒体厚」  

**共性 Job**：  
「把异质材料丢进一块**可缩放、可挪动、可嵌套**的桌面，用空间关系思考，而不是用列表或瀑布流消费。」

## 1. 竞品分层（修订）

| 层级 | 定义 | 代表 | 对 IC2 的竞争强度 |
|------|------|------|-------------------|
| **L0 空间思考画布** | 无限画布 + 卡片/媒体 + 个人深度思考 | **Muse（现 Allume）**、Heptabase（白板侧）、部分 Scrintal | **高**（心智最近） |
| **L1 系统白板** | 系统预装 / 生态默认「自由板」 | **Apple Freeform**、Microsoft Whiteboard、部分 OneNote 自由画布 | **高**（零成本默认选项） |
| **L2 轻量参考摆盘** | 图为主、陪伴创作软件 | PureRef、BeeRef、Kuadro | **中高**（交互参照与迁移用户） |
| **L3 创意项目看板** | 情绪板 + 结构化卡片 + 协作 | Milanote、Whimsical | **中** |
| **L4 团队会议白板** | 工作坊、流程图、大量协作 | Miro、FigJam | **低–中**（场景不同） |
| **L5 本地素材库** | 强检索、弱自由画布 | Eagle、Billfish | **低直接 / 高互补** |
| **L6 宿主内参考** | 画在宿主文档里的 ref | Krita Reference 等 | **低**（跨应用场景仍要独立窗） |

**相对上一版的调整**：

- **去掉** Pinterest / Cosmos 等社交剪藏（与「自由画布组织」不在同一问题空间）。  
- **系统白板从边缘（原 L5）提到 L1**：Freeform 等是「要不要装专用软件」的第一道墙。  
- **Muse 进入 L0**：你参考过其优点；且与「灵感 + 写作 + 思考」方向一致。  
- PureRef 仍重要，但降为 **L2 交互与轻量标杆**，不再是唯一叙事中心。

### 1.1 评价维度

1. **空间自由度**：任意放置、缩放、旋转、叠放、非网格强制  
2. **异质媒体**：图 / 视频 / 文本 / 链接 / 嵌入 / 涂鸦  
3. **结构深度**：分组、嵌套、进入子画布  
4. **写作与研究**：长文本、粘贴、链接预览、对照研判  
5. **本地与所有权**：离线、便携工程、无账号  
6. **轻量与伴随**：启动、内存、置顶  
7. **平台默认性**：是否系统自带 / 生态零摩擦  
8. **商业模式**：免费 / 买断 / 订阅

## 2. 为什么「自由画布」能催生新知（产品哲学）

| 组织方式 | 认知效果 | 代表工具形态 |
|----------|----------|----------------|
| **瀑布流 / 时间线** | 消费、滑动、弱空间关系 | 剪藏流、信息流 |
| **线性文档 / 大纲** | 顺序论证、难并置对照 | 传统笔记 |
| **强制网格 / 相册** | 整齐、难表达「亲疏远近」 | 素材库缩略图 |
| **自由二维画布** | 距离=关系、大小=权重、叠放=层次 | PureRef / Muse / Freeform / **IC2** |
| **嵌套空间** | 主题可「走进去」，全局不爆炸 | IC2 堆栈、部分工具的 frame/group |

你的主张可以写成产品原则：

> **排布即思考。**  
> 当材料以空间关系呈现时，对比、聚类、例外、空白都会变成可见的「想法」。

这同时解释了：为何剪藏流无法替代画布，以及为何 IC2 的嵌套堆栈是结构级差异（不仅是好看）。

## 3. L0 — 空间思考画布（Muse 等）

### 3.1 Muse → Allume（你参考过的方向）

| 项 | 内容 |
|----|------|
| **现名** | App Store 上常见为 **Allume**（formerly Muse / Muse Canvas for Ideas） |
| **平台** | iPad / iPhone / Mac（Apple 生态） |
| **定位** | **Canvas for thinking** — 收集灵感与碎片想法，在空间中理清问题 |
| **内容** | 手写墨迹 + 文本 + 卡片；图、视频、PDF、链接与其它文件可同板 |
| **体验重点** | 无限画布、卡片化、Apple Pencil、私密思考（偏个人深度，而非会议白板） |
| **商业** | 免费 + 内购（以商店为准） |
| **公开叙事** | 介于白板自由与文档结构之间；强调 focus thinking |

**值得借鉴（与 IC2 方向契合）的点**：

1. **「思考工具」而非「参考图工具」** 的品类话术 — 自动跳出纯艺术圈。  
2. **卡片 + 自由空间**：对象有边界感，又可任意摆。  
3. **异质材料同一画布**（不止图）。  
4. **个人深度优先**（可后做协作，不先做会议室）。  
5. **从收集到澄清的闭环**：灵感进来 → 空间整理 → 形成判断/文稿。

**Muse/Allume 相对 IC2 的差异**：

| Muse/Allume | IC2 |
|-------------|-----|
| Apple 生态、触控/笔原生 | 桌面 Win（Mac 规划中）、键鼠 + 快捷键 |
| 手写与卡片思考强 | 键鼠变换（G/R/S）、裁切、视频、链接预览、嵌套堆栈强 |
| 偏「想法澄清」 | 偏「材料厚 + 可工程保存的研究桌」 |
| 订阅/内购云生态可能 | `.icanvas` 本地便携 |

**结论**：Muse 是 **品类与体验参照**，不是功能清单照抄。IC2 可打「桌面端、本地、媒体更厚、可嵌套进入的研究桌」。

### 3.2 其它空间笔记 / 白板笔记（简述）

| 产品 | 备注 |
|------|------|
| **Heptabase** | 卡片白板 + 深度笔记；研究向；偏订阅与知识库 |
| **Scrintal** | 视觉化笔记 |
| **Obsidian Canvas** | 本地库内白板；生态强但媒体摆盘与「桌面产品」感弱 |

对 IC2：说明 **「空间化思考」赛道已成立**，且用户跨研究/写作/产品，不限艺术。

## 4. L1 — 系统白板（竞争强度上调）

系统预装 = **零获客成本 + 默认心智**。用户常问：「系统自带的不够吗？」

### 4.1 Apple Freeform

| 项 | 内容 |
|----|------|
| **定位** | Apple 生态无限白板：头脑风暴、布局、轻协作 |
| **优势** | 免费、预装、iCloud、Pencil、跨 iPhone/iPad/Mac、协作人数多 |
| **短板** | 非「研究级媒体桌」；链接预览/视频/工程文件/嵌套深度、专业变换与桌面伴随（置顶）弱；非 Windows |
| **对 IC2** | **Mac/iPad 用户第一替代选项**；Win 用户无 Freeform → IC2 窗口更大 |

### 4.2 Microsoft Whiteboard / OneNote 自由面

| 项 | 内容 |
|----|------|
| **优势** | Win/365 生态、会议场景、免费档 |
| **短板** | 会议/教育感强；个人灵感与选品 research 的「材料密度」与本地工程弱 |
| **对 IC2** | Win 上的「默认板」噪音；差异靠 **本地工程、嵌套堆栈、链接/视频/裁切、键鼠效率** |

### 4.3 如何赢过系统白板（话术与能力）

| 用户质疑 | 回应方向 |
|----------|----------|
| Freeform 免费啊 | 系统白板偏协作涂鸦；IC2 是 **可保存的研究/灵感工程**（媒体+文字+嵌套） |
| 我只是随便贴贴 | 轻量场景认怂；**材料一多、要对照研判、要带走文件** 时系统板变乱 |
| 要协作 | 系统板/Miro；IC2 主打 **个人深度与本地所有权** |

## 5. L2 — 轻量参考摆盘（PureRef 系）

仍重要：**交互标杆、轻量神话、创作者迁移入口**。但产品叙事不必再「PureRef 克隆」。

### 5.1 PureRef

| 项 | 内容 |
|----|------|
| **定位** | 图片参考无限画布，陪伴绘画/3D |
| **平台** | Win / Mac / Linux |
| **商业** | 可 0 元（pay what you want）→ 装机与心智极强 |
| **优势** | 极轻、置顶/穿透、简单、跨平台、场景文件 |
| **短板** | 媒体偏图；写作/研究/链接/视频/真嵌套弱；品类被钉在「画师 ref」 |

对 IC2：学其 **轻与伴随**；在 **异质材料 + 写作 research + 嵌套** 上拉开。

### 5.2 PureRef 为何「极轻」？底层大致是什么？

公开信息与社区共识（非官方完整白皮书）：

| 层面 | 说明 |
|------|------|
| **技术栈** | **C++ + Qt**（开发者/论坛侧多次确认）。原生编译 GUI，非 Electron/Chromium 壳。 |
| **运行时** | 无浏览器引擎；进程内主要是 Qt 场景图 + 图像解码与纹理。 |
| **功能收敛** | 核心路径极窄：载图、变换、简单注释/裁切、置顶、存场景。**几乎不做** 富文本编辑器、浏览器、协作、插件市场。 |
| **资源策略** | 显示以缩放纹理/缓存为主；不为「全能媒体工作区」常驻解码器与 WebView。 |
| **窗口模型** | 小工具窗 + 始终置顶/点击穿透 → OS 级窗口属性，逻辑简单。 |
| **无账号云** | 无同步服务客户端、无登录态。 |

**「轻」的本质**：

```text
轻 ≈ 原生 UI 工具包（Qt/C++）
    × 极窄功能面
    × 无 Chromium / 无重脚本运行时
    × 无云客户端
    × 媒体类型单一（主要是图）
```

**对比 IC2（Tauri + WebView + React）**：

| | PureRef | IC2 |
|--|---------|-----|
| UI | Qt 原生绘制 | 系统 WebView 渲染 HTML/CSS |
| 逻辑 | C++ | TypeScript + React |
| 体积 | 通常很小 | 小于 Electron，但 **大于** 纯 Qt 小工具 |
| 能力密度 | 低（有意） | 高（视频、链接、嵌入、笔记、嵌套…） |
| 轻量上限 | 极高 | 中高；需靠 **虚拟化、懒加载、少 DOM、控 WebView 数量** 逼近 |

**启示**：

1. 不要用「包体接近 PureRef」当 KPI — **品类不同**。  
2. 要学的是 **功能克制与主路径性能**，不是复刻 Qt。  
3. 若未来要「伴随窗极轻模式」：可考虑 **精简 chrome、限制同时解码视频数、嵌入卸载**，甚至长期 **双模式**（研究桌 vs 轻量置顶条）。  
4. Tauri 相对 Electron 已是正确方向；与 PureRef 比「绝对克」会永远吃亏，应比 **同能力密度下的体感**。

### 5.3 BeeRef / Kuadro

开源或 pay-what-you-want 的轻量 ref 板；强化「可 0 元」市场预期。IC2 若收费需靠 **研究/写作/嵌套** 讲清溢价。

## 11. IC2 的优势与短板（基于代码 + 方向）

### 11.1 优势（真实可感知）

1. **自由画布 + 异质媒体**  
   图 / GIF / 视频 / 笔记 / 自由文本 / 链接预览 / 嵌入 / 涂鸦 — 比 PureRef 厚，比剪藏流真能「摆」。

2. **嵌套可进入堆栈**  
   主题可钻入子画布；适合 research 分层（趋势总览 → 品类 → SKU），也适合灵感「抽屉」。结构差异难被 Freeform 一眼抄走。

3. **桌面键鼠效率**  
   G/R/S、多选组变换、吸附对齐靠拢、裁切 — 适合长时间整理材料的桌面用户。

4. **本地工程 `.icanvas`**  
   可备份、可拷贝、无账号墙；符合研究资料与商业敏感材料（选品）的隐私需求。

5. **粘贴与拖放入口**  
   图文、路径、URL — 贴合「写作 + 搜集」习惯。

6. **技术底座务实**  
   Tauri 小于 Electron；功能密度下仍可控（需持续治性能）。

### 11.2 短板（同样真实）

| 短板 | 影响 |
|------|------|
| **无始终置顶 / 穿透** | 无法像 PureRef 盖在别的软件上「边看边干」 |
| **平台单（Win）** | Muse/Freeform 锁 Apple；创意与研究用户大量在 Mac |
| **手写与触控弱** | 难抢 iPad 深度思考场景（Muse 主场） |
| **写作深度有限** | 有笔记/文本，但非长文编辑器、双向链接、导出长文 |
| **Research 专用能力弱** | 无高亮 PDF、无引用、无表格视图、无「视图切换」 |
| **发现与品牌** | 新品类夹缝，叙事若仍说「PureRef 替代」会错位 |
| **WebView 重量** | 永远难比 Qt 小工具；视频/多 iframe 易重 |
| **迁移** | 无 Freeform/Muse/PureRef 导入 |

### 11.3 一句话优劣

> **强在：本地、自由、异质、可嵌套的研究/灵感桌。**  
> **弱在：伴随性（置顶）、生态默认（系统白板）、Apple 触控思考、长文/PDF 研究深度。**

## 12. 如何从众多产品中突围？

### 12.1 不要打的仗

| 战场 | 对手 | 为何别打 |
|------|------|----------|
| 最轻参考窗 | PureRef | C++/Qt + 极窄功能，体积与心智双赢 |
| 系统默认板 | Freeform | 预装免费 |
| 社交灵感流 | （已排除） | 哲学相反 |
| 企业协作 | Miro | 销售与组织模型不同 |
| 全能素材库 | Eagle | 另一条十年产品 |

### 12.2 建议主战场（与你的方向对齐）

**品类名候选（对内）**：

- 空间化研究桌 / Spatial research desk  
- 本地灵感画布 / Local inspiration canvas  
- 混媒体思考板（强调非纯图）

**对外一句话（示例）**：

> 把截图、链接、视频和文字丢在同一块无限桌面上自由排布、嵌套整理 — 用于灵感、写作草稿与选品/趋势 research。本地文件，不订阅。

### 12.3 突围杠杆（按优先级）

| 优先级 | 杠杆 | 说明 |
|--------|------|------|
| **P0 叙事** | 从「艺术 ref」改成「灵感·写作·research」 | 用户池与媒体报道角度立刻变宽；仍保留画师可用 |
| **P0 结构** | 把 **嵌套堆栈** 讲成核心卖点 | 「文件夹是列表，堆栈是可走进的空间」 |
| **P0 场景模板** | 选品对照板、趋势墙、文稿素材板 | 降低「空白画布恐惧症」；跨行业演示 |
| **P1 平台** | macOS | 否则 Muse/Freeform 用户永远到不了你 |
| **P1 伴随** | 置顶（可选穿透） | 写作/查网页时板仍在 |
| **P1 输入** | 剪藏扩展 / 分享到 IC2 | 研究流：浏览器 → 板 |
| **P2 输出** | 导出长图 / PDF / 大纲 | research 要「交结论」 |
| **P2 轻量模式** | 关嵌入、限视频、极简 UI | 逼近「够轻」而不改栈 |
| **P3** | PDF 批注、双向链接 | 仅当坚定 research 深度时 |

### 12.4 差异化公式（建议锁死）

```text
Freeform 的自由
+ Muse 的「为思考而收集」
+ PureRef 的桌面摆放手感（目标）
+ 嵌套堆栈与本地工程（你已有）
+ 视频/链接/笔记混排（你已有）
− 社交流 − 企业协作 − 重型库
────────────────────────────────
= IC2 的独特位置
```

### 12.5 增长与信任

- **演示视频**：一次真实「选品 research」从粘贴到嵌套结论（比功能列表强）。  
- **便携工程**：强调「一块板一个 `.icanvas` 发给同事也能离线开」（若格式稳定）。  
- **克制 AI 叙事**：空间本身已是认知工具；AI 可后置「整理建议」，勿抢主故事。  
- **Agent / MCP 战略空白**（产品正文见 `PRODUCT.md` §5）：  
  > 还没被打穿的位置：**本地、重媒体、强空间语义、可被 Agent 稳定读写的板**——这正是 **IC2 + MCP** 可能站的地方。  
  多数画布 AI 是锦上添花；优先外部 Agent 经 MCP 读写板契约，而非板内全能 Chat。  
- **Win 先做深再 Mac**：或 Mac 优先若目标用户在 Apple — 与 Muse 对打需 Mac。

## 13. 与「剪藏流」的关系（为何排除但仍要懂）

Pinterest / Cosmos 等：

- 强在发现与消费；  
- 弱在 **你无法用距离与层次表达判断**；  
- 内容形态偏图卡。

**策略**：不做竞品列表里的对手；做 **单向进口**（以后扩展、拖图）。用户从剪藏「救出」有价值的材料，在 IC2 里二次空间化 — 这是补完而非替代。

## 15. 结论

1. **定位应上移**：从「艺术 PureRef 替代」→ **本地空间化灵感 / 写作 / research 桌**；艺术只是垂直之一。  
2. **L0 看 Muse，L1 防 Freeform，L2 学 PureRef 之轻与伴随，不学其品类钉死。**  
3. **已有优势**：异质媒体、嵌套堆栈、桌面变换、本地工程 — 与哲学一致。  
4. **战略空白（AI 时代）**：本地 + 重媒体 + 强空间语义 + **Agent 可稳定读写** → **IC2 + MCP**（详见 `PRODUCT.md` §5）；勿用板内套壳生成抢主故事。  
5. **致命缺口**：叙事、Mac、置顶、从收集到「输出结论」的闭环；Board 操作契约尚未对外暴露。  
6. **PureRef 极轻** = C++/Qt + 极窄功能 + 无 Web 运行时；IC2 应追求 **同场景体感足够轻**，而非同架构同体积。  
6. **突围**：锁死「自由空间催生新知」；用嵌套 + 混媒体 + 本地；场景化模板；别进社交流与企业白板红海。
