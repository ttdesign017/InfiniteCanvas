> **状态**：本地产品文档（`docs/` 默认不入库）  
> **基线**：v1.1.0 · Tauri 2 · Windows · `.icanvas`  
> **关联**：竞品与分层见 `COMPETITIVE_RESEARCH.md`；工程化门槛见 `ENGINEERING_PRODUCTIZATION.md`；功能 backlog 见 `feature requests.md`。

---

## 1. 一句话定位

> **本地优先的空间化灵感与研究桌面**  
> Spatial canvas for inspiration, writing fragments, and research — free layout, nested structure, mixed media.

绘画参考（PureRef 主场景）是相邻能力，**不是唯一北极星**。  
Muse 一类 thinking canvas、Freeform 一类系统白板，与本产品**同样相关甚至更相关**。

---

## 2. 用户要完成的 Job

把异质材料丢进一块**可缩放、可挪动、可嵌套**的桌面，用**空间关系**思考，而不是用列表、文件夹树或瀑布流消费。

| 维度 | 方向 |
|------|------|
| **灵感搜集** | 图、视频、音频、链接、摘录、网页嵌入，随手丢上画布 |
| **写作 / 混排** | 笔记卡、自由文本、图文同屏 |
| **Research / 研判** | 选品、趋势、对照、聚类；邻近与层次代替目录树 |
| **组织哲学** | 自由二维空间：距离≈关系，大小≈权重，叠放≈层次，嵌套≈主题可进入 |

### 典型场景

- **选品 / 电商**：详情截图、价格带、竞品视频、利弊笔记并置  
- **研究 / 咨询 / 趋势**：多来源摘录、图表、链接卡同一视野  
- **写作 / 内容**：素材碎片 + 段落草稿的空间化大纲  
- **设计 / 品牌**：情绪板，但带文字与链接深度  
- **个人项目桌**：混合一切，比笔记更「看见全局」，比会议白板更「媒体厚」

## 3. 明确不做什么（边界）

| 不做 / 弱做 | 原因 |
|-------------|------|
| 社交剪藏流（Pinterest / Cosmos 类） | 瀑布流与弱空间关系，与哲学冲突 |
| 企业实时协作白板（Miro 主场） | 另一类产品与组织形态 |
| 完整 DAM 素材库（Eagle 级） | 检索与库规模工程巨大；可互补 |
| 宿主内绘画取色工具链 | 可后置，非核心 |
| 板内全能 Chat / 「又一个 AI 壳」 | 勿抢「空间即思考」主故事（见 §5） |

## 4. 产品原则

1. **排布即思考** — 空间关系是一等公民，不是装饰。  
2. **本地优先、板即工程** — `.icanvas` 可携带、可离线；无账号不是残缺，是立场。  
3. **媒体要厚** — 图 / 视频 / 音频 / 链接 / 嵌入 / 笔记同桌，而非「只能贴纸」。  
   - 视频支持全分辨率帧快照（Shift+C）与逐帧步进；系统剪贴板可导出图/文到外部应用。  
4. **结构可进入** — 嵌套 stack：主题能「走进去」，全局不爆炸。  
   - 涂鸦是 free-canvas 墨迹层：不参与折叠 fan / folder 几何，退出 stack 时快速消失。  
5. **伴随创作** — 轻启动、可沉浸；置顶/透明度等可演进，但不牺牲主路径清晰度。  
6. **AI 克制、可后置** — 空间本身已是认知工具；AI 是顾问或外脑，不是机长（详见 §5）。

## 5. 战略空白与 AI / Agent 定位

### 5.1 行业现状（摘要）

协作白板、笔记、系统白板在「思考画布」上有品牌与分发；多数「画布 + AI」是 **锦上添花**（生成 sticky、总结、聚类），真正 AI-native 的往往是 Chat/Agent，画布只是输出表面。

### 5.2 还没被打穿的位置（IC2 可站之处）

> **还没被打穿的位置：本地、重媒体、强空间语义、可被 Agent 稳定读写的板——这正是 IC2 + MCP 可能站的地方。**

拆开四条，每条都是筛选器：

| 维度 | 含义 | 多数竞品弱点 |
|------|------|----------------|
| **本地** | 离线、隐私、板文件在用户磁盘；无强制云账号 | 云白板 / 订阅笔记默认上云 |
| **重媒体** | 真实参考图、视频、音频、链接预览，而非仅彩色贴纸 | 会议白板偏便签与形状 |
| **强空间语义** | 自由位姿、嵌套容器、fan/gather、用户手摆的关系有意义 | 自动网格/时间线冲掉语义；AI 乱排会毁掉思考 |
| **可被 Agent 稳定读写** | 结构化 board 模型 + 明确工具契约（读树、读写卡片、建议坐标），而非只能截图给模型 | 板是像素海滩，Agent 只能「看图说话」不能可靠操作 |

**合在一起**才构成空白：  
只做本地 ref（如 PureRef）→ Agent 难读写；  
只做云 AI 白板 → 非本地、媒体与空间语义常被协作模板化；  
只做 MCP 空壳 → 没有值得操作的厚桌面。

**IC2 已有前三条的产品胚子**（本地 `.icanvas`、混媒体、嵌套 stack）。  
**第四条是战略延伸**：把板暴露为 **Agent 可操作的空间工作台**（优先 MCP），而不是先做板内套壳生成。

### 5.3 AI 产品原则（落地）

| 原则 | 说明 |
|------|------|
| **空间与本地是主角** | 主故事仍是「桌」；AI 不抢导航与手感 |
| **优先 MCP / 外脑** | Agent 在外（用户自选模型与 Key）；IC2 提供稳定读写契约 |
| **内嵌 API 可薄、可晚** | 选区总结成新卡等 OK；默认不做「一键 AI 重排整板」 |
| **写入可撤销** | 任何自动变更必须进 history / 可 Undo |
| **多模态优先理解已有材料** | 打标、对照、摘要已有图与链接，优于堆生成假图 |
| **先 Board Operations，再接 AI** | 统一 list / get / patch / create / layout-suggestion；MCP 与内嵌 UI 共用 |

### 5.4 建议演进顺序

```
Board 操作层（结构化读写，工程化）
        │
        ├─► MCP Server（外部 Agent 连接）     ← AI 主路径
        │
        └─► 可选薄 UI（内嵌 1～2 个确认型能力） ← 锦上添花、复用同一 API
```

| 阶段 | 做什么 | 不做 |
|------|--------|------|
| 现在 | 板子可靠、空间语义稳定；**board-ops 领域层已就绪**（见 `BOARD_OPS.md` / `MCP.md`） | 不急着板内 Chat |
| AI-1 | 只读 MCP：树、选区、导出文本 | 不开放无确认的批量删除 |
| AI-2 | 受限写入：新增笔记/标签建议 | 不默认自动打乱坐标 |
| AI-3 | 建议排布（人确认 + Undo） | 不对标 Miro 全能 Assist 叙事 |

### 5.5 对外怎么说（叙事）

- **主叙事**：本地空间化灵感与研究桌面。  
- **副叙事（Agent 时代）**：这块桌还能被你的 Agent 稳定读写——材料仍在你磁盘上，思考关系仍是空间的。  
- **避免**：「AI 无限画布」「一键生成整板」等与巨头同质化口号。

## 6. 与竞品心智的相对位置（极简）

```
Freeform 的自由
+ Muse 的「为思考而收集」
+ PureRef 的桌面摆放手感（目标）
+ 嵌套堆栈与本地工程（已有）
+ 视频/链接/笔记混排（已有）
+（战略）Agent 可读写的板契约（MCP 方向）
− 社交流 − 企业协作 − 重型库 − 板内 AI 抢戏
────────────────────────────────
= IC2 的独特位置
```

分层与细表见 `COMPETITIVE_RESEARCH.md`。

## 7. 成功怎么量（产品侧，非工程）

| 层级 | 信号 |
|------|------|
| **自用** | 真实 research / 写作 / 选品任务会**下意识打开** IC2 |
| **Agent** | 至少一条工作流：外脑读板或写卡，比纯聊天更省事 |
| **对外** | 陌生人装上后，10 分钟内能完成「丢材料 → 排布 → 存盘再开」 |
| **不做虚荣指标** | 功能清单长度、是否勾选「有 AI」 |
